# RSS-Twitter-Viewer V3.0

## 项目简介

轻量化纯前端 RSS 阅读器，部署于 Cloudflare Pages，仅个人自用，内置代理转发，专门适配 X (Twitter) RSS 订阅，解决海外资源跨域加载、视频图片缓冲慢问题；采用本地缓存优先机制，打开页面秒加载历史内容，仅点击主页静默后台检测更新，滚动分页 50 条分批渲染，大幅降低网络等待耗时。

## 核心特性 V3.0 更新汇总

### 一、交互逻辑重大优化（回归旧版极速浏览体验）

1. **本地缓存优先渲染**
   * 页面初始化优先读取本地缓存文章 2\~3 秒直接展示，不阻塞等待全量 RSS 网络请求；
   * 仅首次打开页面完整拉取一次 RSS 存入本地缓存，后续切换页面复用缓存数据。
2. **增量更新静默检测**
   * 点击【主页】仅后台异步对比 RSS，页面原有内容不销毁、不刷新；
   * 检测到新内容：顶部弹出蓝色提示条「检测到 XX 条新内容，点击刷新」，点击后才全量更新页面；
   * 无新内容：浅灰提示条 2.5 秒自动消失，不干扰当前浏览阅读。
3. **滚动分页加载**
   * 固定单页渲染 50 条内容，滚动到底部自动追加下 50 条，不会一次性加载全部媒体资源；
   * 切换【管理订阅源】【我的收藏】不再强制全量重载所有 RSS，复用本地缓存数据。

### 二、媒体加载 & 播放提速优化

1. 视频预加载策略修改：`preload="none"`，页面加载完全不下载视频，仅点击播放才发起请求，大幅减少首屏加载带宽与等待时长；
2. 修复推特视频封面 poster 预览图空白问题：封面图片自动匹配媒体代理，跨域正常加载缩略图；
3. 媒体代理开启分片流式传输、24 小时持久浏览器缓存，重复打开图片 / 视频几乎无缓冲等待；
4. 仅 X (Twitter) 域名图片 / 视频走 CF 媒体代理，国内图文资源浏览器原生直连，无中转延迟。

### 三、导出功能全平台兼容修复（重点修复）

1. 重写订阅一键导出函数，双方案降级兼容：
   * 电脑 Chrome/Edge：标准 Blob 下载；
   * iPhone、安卓、Via 浏览器：自动切换 Base64 DataURL 兜底方案；
2. 彻底解决移动端报错：`blob:协议拦截`、`无法下载非http资源`，手机端稳定导出 json 备份；
3. 导入功能不变，支持本地 json 备份文件恢复订阅列表。

### 四、网络代理分流规则（固定不变）

1. RSS 文本抓取：仅`twitter.com/x.com` RSS 链接走`/rss-proxy`代理，国内 RSS 浏览器直连；
2. 媒体资源：仅`video.twimg.com/pbs.twimg.com`图片、视频走`/media-proxy`代理；
3. 页面 DOM、文字、UI 渲染完全本地浏览器处理，无任何代理转发。

## 部署结构

plaintext

```
rss-twitter-viewer/
├── index.html               # 主页面 V3.0 完整前端代码
└── functions/
    ├── rss-proxy.js         # RSS文本跨域代理，5分钟边缘节点缓存
    └── media-proxy.js       # 图片/视频媒体代理，24小时持久缓存+分片续传
```

## 部署步骤（Cloudflare Pages + GitHub CodeSpace）

1. 新建 GitHub 仓库，上传项目完整文件，目录结构严格保持如上；
2. Cloudflare Pages 绑定该仓库，构建命令为空，输出目录填 `/`；
3. 等待自动部署完成，访问分配的`*.pages.dev`域名即可使用；
4. 如需自定义域名，在 Cloudflare 后台绑定已备案域名。

## 使用说明

### 基础操作

1. **添加订阅**：顶部【添加订阅】，填写订阅名称、资源类型、RSS 链接，确认添加；
2. **浏览主页**：打开自动展示本地缓存文章，滚动到底部加载更多；点击主页静默检测更新；
3. **单订阅单独查看**：【管理订阅源】→ 对应订阅点击【查看】，仅筛选该订阅内容，不重载全部 RSS；
4. **收藏文章**：卡片右下角点击收藏，收藏内容统一在【我的收藏】分页查看；
5. **导入 / 导出备份**：【管理订阅源】顶部按钮，一键导出全部订阅 json 备份，本地文件可导入恢复。

### 媒体相关

1. 图片：自动懒加载，可视区域外不请求资源，点击图片进入全屏预览；
2. 视频：页面加载不预下载，点击播放按钮才缓冲，支持进度拖动、分片续播；推特视频自带封面缩略图。

## 本地存储说明

所有订阅列表、收藏文章均存储在浏览器`localStorage`，仅保存在当前设备，无云端上传，隐私安全；清除浏览器数据会清空订阅与收藏，操作前建议先一键导出备份。

## 配套代理配置说明

1. `rss-proxy.js`：RSS XML 文本代理，缓存 300 秒，缩短重复刷新等待；
2. `media-proxy.js`：媒体资源代理，支持 Range 分片请求、24 小时不可变缓存，解决视频长时间缓冲；
3. 仅放行推特官方媒体域名，其余图片 / 视频直接 302 重定向原生链接，不占用代理带宽。
